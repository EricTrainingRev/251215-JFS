# P1 Presentation Requirements

## General Requirements
- All team members need to participate in presentation
- Presentations should be ~ 15 minutes long
- A Test report listing all tests and defects discovered should be generated the day before presentations

## Presentation Order of Operations
1. Introduction
    - Introduce team
    - Briefly explain what work was accomplished this Sprint (high level overview)
    - Provide overview of presentation content
2. Product Walkthrough
    - Showcase all tests
        - do not showcase source code at this time: show the automated testing in action
    - Showcase work done on stretch goals
3. Mini Retrospective
    - Describe what can be improved upon during the next sprint
    - Describe what went well during the current sprint
